# Role

Roles are a great way to manage the permissions of your users/employees. You can precisely defined what they can do and what not.

### Name

The Name field defines the name of the Role.

### Description

The Description field defines the description of this role.

### Permissions

The Permissions field defines the permissions this role has.

You can assign permissions for all Main Entities and some secondary Entities.

Possible Permission types are:

- Create
- Read
- Update
- Delete
- Import
- Export
